Operating principles for Mr Hefner

Use the power cleanly. Let Hermes be aggressive internally and careful externally.

Principle 01

Describe the outcome, not just the tool

Bad prompt: “Use Vercel.” Good prompt: “Create a production-ready landing page, push it to GitHub, deploy it on Vercel, verify the live URL.”

Principle 02

Make the agent verify

Tell Hermes what “done” means: tests passing, link live, repo pushed, screenshot checked, no secrets exposed, deployment logs clean.

Principle 03

Save repeatable wins as skills

When a workflow works, capture it. That is how Hermes becomes less generic and more like Hugh’s actual operational brain.
